# SkyproWallet

_Учебный проект по frontend-разработке_

SkyproWallet — веб-приложение для учета, анализа и визуализации личных расходов с возможностью фильтрации данных, просмотра статистики и управления расходами за выбранный период.

Проект разрабатывается командой студентов в рамках обучения веб-разработке.

**Цель проекта** — создать удобный и функциональный инструмент для управления расходами.

## Функциональность

### Авторизация

- Вход в приложение
- Регистрация нового пользователя
- Выход из системы

### Учет расходов

- Просмотр списка расходов
- Добавление новых расходов
- Удаление расходов

### Аналитика

- Отображение общей суммы расходов
- Визуализация распределения расходов по категориям за выбранный период

### Календарь

- Прокрутка календаря
- Выбор периода просмотра данных
- Отображение расходов за выбранный период

### Пользовательский опыт

- Отображение состояний загрузки
- Обработка ошибок ввода

## Установка проекта

Клонируйте репозиторий:

```bash
git clone https://github.com/kors-ak/skypro-wallet.git
```

Перейдите в папку проекта:

```bash
cd skypro-wallet
```

Установите зависимости:

```bash
npm install
```

Запустите проект:

```bash
npm run dev
```

## Организация работы

- Управление задачами ведется в [Kaiten](https://skywallet.kaiten.ru/space/795785/boards?focus=board&focusId=1795245).
- Основная ветка разработки — `dev`.
- Каждый участник работает в собственной ветке Git.
- Перед созданием новой рабочей ветки необходимо синхронизироваться с актуальной версией ветки `dev` (`git pull origin dev`).
- Все изменения отправляются через Pull Request в ветку `dev`.
- Капитан команды проводит ревью.
- После проверки изменения объединяются с веткой `dev`.
- По завершении спринта создается Pull Request из ветки `dev` в ветку `main`.
- После проверки наставником ветка `dev` объединяется с веткой `main`.

## Git Workflow

### Ветки

- `main` — стабильная версия проекта
- `dev` — основная ветка разработки

Примеры рабочих веток:

- `feature/registration-form`
- `bug/login-validation`
- `hotfix/auth-crash`

Создание ветки:

```bash
git checkout dev
git pull origin dev
git checkout -b /название ветки/
```

### Коммиты

Название коммита должно кратко и понятно описывать внесенные изменения.

Примеры:

```bash
git commit -m "feat: Добавляет форму регистрации"
git commit -m "fix: Исправляет валидацию поля email"
git commit -m "style: Обновляет стили страницы аналитики"
git commit -m "refactor: Рефакторит компонент календаря"
```

### Соглашение по коммитам

- `feat` — новый функционал
- `fix` — исправление ошибок
- `refactor` — рефакторинг кода
- `style` — изменения форматирования и стилей
- `docs` — изменения документации

### Code Review

Перед отправкой Pull Request необходимо:

- Убедиться, что проект запускается без ошибок.
- Проверить отсутствие предупреждений линтера.
- Убедиться, что код соответствует стилю проекта.
- Самостоятельно протестировать реализованный функционал.

### Отправка изменений

После завершения работы отправьте изменения в удаленный репозиторий:

```bash
git add .
git commit -m "feat: /внесенные изменения/"
git push origin /название ветки/
```

### Pull Request

Перед созданием PR:

- проверьте работу приложения;
- убедитесь в отсутствии конфликтов с `dev`;
- добавьте описание изменений и ссылку на задачу в Kaiten.

Описание Pull Request должно содержать:

- краткое описание реализованного функционала или исправления;
- плановое и реальное время выполнения;
- ссылку на задачу в Kaiten.

После проверки и одобрения Pull Request изменения могут быть объединены с веткой `dev`.
