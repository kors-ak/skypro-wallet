# SkyproWallet

_Учебный проект по frontend-разработке_

SkyproWallet — SPA для учета, анализа и визуализации личных расходов.

Пользователь может зарегистрироваться, вести учет транзакций, анализировать расходы за выбранный период, просматривать статистику по категориям и управлять своими данными через удобный адаптивный интерфейс.

Проект разработан командой студентов в рамках обучения frontend-разработке.

**Цель проекта** — создать удобный и функциональный инструмент для управления расходами.

## Функциональность

### Авторизация

- Вход в приложение
- Регистрация нового пользователя
- Выход из системы
- Защищенные маршруты

### Учет расходов

- Просмотр списка расходов
- Добавление новых расходов
- Удаление расходов

### Аналитика

- Просмотр общей суммы расходов
- Диаграмма распределения расходов по категориям
- Отображение данных за выбранный период
- Автоматический пересчет статистики

### Календарь

- Прокрутка календаря
- Выбор периода просмотра данных
- Отображение расходов за выбранный период

### Пользовательский опыт

- Адаптивная верстка
- Состояния загрузки
- Toast-уведомления
- Обработка ошибок API
- Страница 404

## Дизайн

Интерфейс реализуется на основе предоставленного [макета в Figma](https://www.figma.com/design/G6NqO7gpXDOS6ahqdBe0JB/Skypro.Wallet?node-id=1-4) с максимальным соблюдением визуального оформления и пользовательских сценариев.

## Технологии

### Основной стек

- HTML5
- CSS3
- JavaScript (ES6+)
- React
- Vite

### Библиотеки

- React Router DOM — маршрутизация приложения
- Axios — работа с API
- Styled Components — стилизация компонентов
- Recharts — визуализация аналитики
- SimpleBar — кастомный скролл
- Sonner — toast-уведомления

### Инструменты разработки

- ESLint
- Prettier
- Git
- GitHub

## Архитектурные решения

- SPA на React
- Маршрутизация через React Router
- Глобальное состояние реализовано с помощью Context API
- Работа с API вынесена в отдельный слой сервисов
- Логика авторизации отделена от логики работы с расходами
- Компоненты разделены на независимые функциональные модули
- Переиспользуемые UI-компоненты вынесены в отдельные директории

## Установка проекта

Клонируйте репозиторий:

```bash
git clone https://github.com/kors-ak/skypro-wallet.git
```

Перейдите в папку проекта:

```bash
cd skypro-wallet
```

Установите зависимости:

```bash
npm install
```

Запустите проект:

```bash
npm run dev
```

Сборка проекта:

```bash
npm run build
```

## Организация работы

- Управление задачами велось в Kaiten.
- Основной веткой разработки являлась `dev`.
- Каждый участник работал в собственной Git-ветке.
- Все изменения проходили code review.
- После проверки изменения объединялись в ветку `dev`.
- После завершения разработки ветка `dev` объединялась с `main`.
